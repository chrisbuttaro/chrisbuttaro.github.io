Coding with Java

Equals

=   == and .equals() serve three distinctly different functions in Java. = is an assignment operator, == is a relationship operator, and .equals() is a method for testing the equality of objects.

= is used to assign a value from an operand on the right to an operand on the left. One example of this variable declaration.


int j=10;
boolean y=true;

A more straight forward example is changing a variable's value within a script.

j=20;
y=false;

In both examples the value from the right is assigned to the operator on the left.

== is a relational operator that tests the relationship of two operands specifically to see if they are equal. Relational operators return a boolean value of true or false.

if (x==y){
executeFunction();
}

In this case if x and y have the same value the expression in parens will return true and the function will execute. Otherwise the expression will return false and the compiler will skip over the if statement.

.equals() is a method which compares two objects to see if they are the same. If the objects have the same state and behavior they are considered to be the same object and .equals() will return true. If they are different objects .equals() will return false. One use of .equals() is to compare two String objects.

String x="A";
String y="A";
String z="B";
x.equals(y) //will return true
x.equals(z) //will return false
